# ゲームシステム概要メモ

最終更新: 2026-02-24

## 1. 画面構成
- `story.html` を Vue (`StoryApp.vue`) でマウントし、各セクションを読み込む構成。
- 主な表示領域:
  - 左上: キャラクターカード (`characterInfo.js`)
  - 右上: ステータス/装備/戦闘タブ (`tabContainer.js`)
  - 下部: スキル一覧・スキルセット・条件Pなど (`skillContainer.js`)
- 主要モーダル:
  - `DetailModal.vue`（アイテム/スキル詳細）
  - `SkillSetModal.vue`（スロットセット）
  - `ConditionPassiveModal.vue`（条件P一覧）
  - `EquipCompareModal.vue`（装備比較）

## 2. キャラデータと保持状態
- キャラデータは API 経由で取得して `playerData` に保持。
- 複数キャラを `characterList` で管理。
- キャラ切替時にキャラ別で保持される状態:
  - 選択スキル (`selectedSkillsByCharacter`)
  - 攻撃手段 (`selectedAttackOptionByCharacter`)
- アイテム状態は `装備中 / 持ち物 / 倉庫` を正規化して管理。

## 3. スキル枠と分類
- 戦闘用スロット順: `A, S, Q1, Q2, M`
- 分類ごとの代表:
  - A: 通常攻撃系
  - S/Q: スキル攻撃系
  - M: 魔法
  - P: パッシブ（条件付き含む）
- スキルセット時はスロット候補を制限し、モーダルから選択。

## 4. 戦闘計算（共通）
- 基本倍率:
  - `判定倍率 = 1 + 攻撃判定値 / 100`
  - `追加威力倍率 = 1 + 追加威力値 / 500`
  - `基本倍率 = 判定倍率 × 追加威力倍率`
- 魔法（M）では魔法倍率を追加:
  - `魔法倍率 = 0.15 + (魔法Lv/7) × 0.20`（M枠計算）
- 全力ON時の補正:
  - 各スキル全力: `全力/100 + 1.15`
  - A枠全体全力: `全力合計/100 + 1.25`
- 全威力倍率:
  - 各スキルの `全威力` を乗算して最終倍率に反映。

## 5. 通常攻撃（A枠）と攻撃手段
- A枠の通常攻撃判定対象:
  - `切断 / 貫通 / 打撃 / 防御 / 射撃 / 狙撃 / 攻勢盾`
  - または属性が `通常攻撃`
- 通常攻撃時は `通常攻撃の値 + 攻撃手段データ（武器/肉体）` を合算して計算する。
- 現在仕様:
  - `威力` は攻撃手段ベースのみ（属性値は威力に加算しない）
  - `属性` は属性欄で独立表示
- 攻撃手段初期選択:
  - 武器があれば武器優先
  - 武器がない場合は `素手` と `爪` 比較（爪は `威力×0.9` で比較）

## 6. 攻撃回数
- A枠通常攻撃時:
  - 攻撃手段の攻撃回数 + 条件Pの攻撃回数補正を基準にする。
- S/Q枠は表示/合計時に分割ロジックあり（M枠は合計除外）。
- 「倍化」「3倍化」などは攻撃回数倍率として適用。

## 7. Cr・ダメージブレ・貫通・守り
- Cr系の集計:
  - `A + S~Q` の合計
  - A未設定時は基礎値 `Cr率10 / Cr威力130`
  - Aが通常攻撃時は `通常攻撃の値 + 攻撃手段データ` を合算して参照
- ダメージブレ:
  - 基礎 `40~100`
  - 最低ダメージ合計で下限のみ上昇
  - 表示: `(40 + 最低ダメージ合計) ~ 100`
- 貫通:
  - `防御貫通` と `魔法貫通` を別で扱う（表示時に統合表記）
- 守り:
  - `物理ガード` と `魔法ガード` の両方がある場合は、高い方の値を参照して表示する。
  - `物理ガード` / `魔法ガード` は、それぞれのガード値に攻撃手段の防御値を加算して決定する。
  - `魔法ガード` は `S〜Q` 側に `+1` 以上の寄与がない場合、`0` 扱いになる。

## 8. 条件P（条件付きパッシブ）
- 条件キー:
  - `攻撃手段 / 条件スキル(条件) / 条件属性`
- 複数条件は AND 判定。
- 条件一致したPは計算へ反映。
- `攻撃手段:肉体` は通常攻撃向けの特例処理あり。
- 一覧や説明には `P件数` 表示を出し、モーダルで一致Pを確認可能。

## 9. 装備・持ち物・倉庫
- 装備変更は比較モーダルで遷移先を選択可能。
- 空きスロットは `空き` として扱う。
- 装備後は関連表示（ステータス、攻撃手段、選択スキル表示）を再描画。

## 10. ダイスとログ
- 戦闘画面でダイス個数/面数を指定してロール。
- ログはキャラ別管理と全体管理の切替を想定。
- 送信データの表示や API 連携ログ整形（Excel貼り付け向け）対応あり。

## 13. スキルセット計算手順（確定メモ）
1. A〜Mのステータス・技能補正を先に合算する。
2. その時点で A〜M それぞれの倍率情報を保持する。
3. A〜M 各スロットの威力を、`追加威力` と `判定` と各種倍率から算出する。
4. 3の結果に対して `全威力` の倍率を適用する。
5. この結果を、A〜M 各スロットのツールチップ表示データとする。
6. さらにここから `威力 / 守り / 状態` を A側・M側それぞれで合計表示する。
7. 最後に `A合計値` 枠では、A側の各項目合計を合算して表示する。
8. 守り表示では `物理ガード / 魔法ガード` の高い方を参照し、魔法ガードは `S〜Q` 側寄与が `+1` 未満なら `0` とする。
